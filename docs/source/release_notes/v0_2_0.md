# v0.2.0

AutoLyap 0.2.0 is a substantial update focused on API clarity, backend
flexibility, new documentation, and stronger test coverage.
Status: Unpublished.

## Highlights

- Added top-level {py:class}`SolverOptions <autolyap.SolverOptions>` support to
  configure solver backends explicitly in analysis calls.
- Added end-to-end documentation in {doc}`../quick_start`.
- Added worked examples and convergence assets for
  {doc}`../examples/define_your_own_algorithm/proximal_gradient_method`,
  {doc}`../examples/proximal_point`,
  {doc}`../examples/heavy_ball/index`,
  {doc}`../examples/nesterov_fast_gradient`, and
  {doc}`../examples/douglas_rachford/maximally_monotone_plus_strongly_monotone_lipschitz`.
- Improved problem-class validation and error messages.
- Updated Lyapunov SDP search routines to return certificates
  in solver results.
- Standardized Lyapunov solver result payloads to explicit status dictionaries
  (`status`, `solve_status`, `rho`/`c_K`, `certificate`) and removed
  legacy boolean success signaling.
- Added protocol-based backend typing in
  {py:mod}`autolyap.utils.backend_types` and propagated structural typing
  through CVXPY/MOSEK-facing internals for stronger static checks.
- Added `verbosity` controls for Lyapunov search routines:
  - {py:meth}`autolyap.IterationIndependent.search_lyapunov`
  - {py:meth}`autolyap.IterationIndependent.LinearConvergence.bisection_search_rho`
  - {py:meth}`autolyap.IterationDependent.search_lyapunov`
- Set Lyapunov search defaults to `verbosity=1` for concise diagnostics.
- Added runtime diagnostic summaries covering:
  - nonnegativity-constrained scalars,
  - PSD-constrained matrices (minimum-eigenvalue checks),
  - equality constraints (`max_abs_residual`, `l2_residual`).
- Added project citation metadata and citation guidance.
- Added a dedicated {doc}`../theory` section with detailed mathematical
  formulation pages (notation, interpolation assumptions, and Lyapunov SDP
  constructions).
- Improved math rendering and reference coverage in theory documentation.

## Breaking changes

- Interpolation index tokens were renamed:
  - `i<j` -> `r1<r2`
  - `i!=j` -> `r1!=r2`
  - `i` -> `r1`
  - `i!=star` -> `r1!=star`
- Lyapunov entry points were renamed:
  - `IterationIndependent.verify_iteration_independent_Lyapunov` ->
    `IterationIndependent.search_lyapunov`
  - `IterationDependent.verify_iteration_dependent_Lyapunov` ->
    `IterationDependent.search_lyapunov`
- Deprecated Lyapunov alias methods were removed:
  - `IterationIndependent.verify_iteration_independent_Lyapunov`
  - `IterationDependent.verify_iteration_dependent_Lyapunov`
- Lyapunov API return types changed from scalars/booleans to result
  dictionaries:
  - {py:meth}`autolyap.IterationIndependent.search_lyapunov`
    now returns `{"status", "solve_status", "rho", "certificate"}`.
  - {py:meth}`autolyap.IterationIndependent.LinearConvergence.bisection_search_rho`
    now returns `{"status", "solve_status", "rho", "certificate"}`.
  - {py:meth}`autolyap.IterationDependent.search_lyapunov`
    now returns `{"status", "solve_status", "c_K", "certificate"}`.
- Legacy `success` booleans were removed. Use `status == "feasible"` to
  detect certified success.
- Result-status semantics are now explicit:
  - `status="feasible"` when a certificate is returned.
  - `status="infeasible"` for genuine infeasibility.
  - `status="not_solved"` for solver/interface errors or indeterminate
    backend statuses (with details in `solve_status`).

## Migration from 0.1.0

1. Replace old interpolation-index literals with the new tokens listed above (for example, `i!=j` -> `r1!=r2`).
2. Replace old Lyapunov entry-point names:
   - `IterationIndependent.verify_iteration_independent_Lyapunov` -> `IterationIndependent.search_lyapunov`
   - `IterationDependent.verify_iteration_dependent_Lyapunov` -> `IterationDependent.search_lyapunov`
3. Update result handling to consume dictionary outputs
   (`result["status"]`, `result["solve_status"]`,
   `result["rho"]`/`result["c_K"]`, and `result["certificate"]`) instead of
   boolean/tuple returns. Treat only `status == "feasible"` as certified
   success and handle `status == "not_solved"` separately from infeasibility.
4. If needed, switch to explicit backend configuration with
   {py:class}`SolverOptions <autolyap.SolverOptions>`.
5. Re-run your analyses and tests to confirm expected rates/certificates.

Example migration for result handling:

```python
# 0.1.0 style:
# ok = IterationIndependent.verify_iteration_independent_Lyapunov(...)

# 0.2.0 style:
result = IterationIndependent.search_lyapunov(...)
if result["status"] == "feasible":
    rho = result["rho"]
    certificate = result["certificate"]
```

Example backend configuration:

```python
from autolyap import SolverOptions

solver_options = SolverOptions(backend="mosek_fusion")  # requires `autolyap[mosek]`
# or:
# solver_options = SolverOptions(backend="cvxpy", cvxpy_solver="CLARABEL")
```

## Documentation and quality improvements

- Introduced documentation navigation, API organization, and quick-start flow.
- Updated diagnostics wording to use neutral phrasing when no violations are
  present (for example, "smallest ... (nonnegative)").
- Added inline clickable math labels (`C1`-`C4`) in theory pages and aligned
  their styling with equation-tag links.
- Clarified theory exposition by centralizing shared solution notation and
  tightening theorem naming/cross-references between SDP and Lyapunov pages.
- Expanded developer internal docs inventories to include package-level
  exports and backend typing helpers.
